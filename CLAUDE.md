# Universal Log Analyzer 1C

## 1. Purpose
Анализатор логов платформы 1С.  
Вход: файлы логов.  
Выход: структурированный отчет в JSON и HTML, содержащий события, проблемы и рекомендации.

---

## 2. Supported Log Types
rphost  
rmngr  
ras  
dbgsrv  
1cv8.log  

---

## 3. MVP Functional Scope
- Автоматическое определение типа лога.  
- Разбор строк: время, уровень, компонент, событие, контекст.  
- Классификация событий: ошибки, предупреждения, медленные SQL, deadlock, долгие транзакции, проблемы памяти.  
- Формирование JSON‑отчета.  
- Формирование HTML‑отчета (минимальная разметка).  
- Генерация рекомендаций по правилам.

---

## 4. Architecture

### 4.1 Modules
Core.Detector — определение типа лога.  
Core.Parser — разбор строк.  
Core.Classifier — классификация событий.  
Core.KnowledgeBase — правила и рекомендации.  
Report.JSONBuilder — генерация JSON.  
Report.HTMLBuilder — генерация HTML.  
UI.Form — минимальный интерфейс выбора файла и запуска анализа.

### 4.2 Processing Pipeline
1. Пользователь выбирает файл.  
2. Detector определяет тип.  
3. Parser преобразует строки в структуру.  
4. Classifier присваивает категории.  
5. KnowledgeBase формирует рекомендации.  
6. Report‑модули создают итоговые отчеты.

---

## 5. Classification Rules (MVP)
ERR → ошибка.  
WARNING → предупреждение.  
SQL длительность > 500 мс → медленный запрос.  
Содержит Deadlock → блокировка.  
COMMIT > 2000 мс → долгая транзакция.  
Memory выше порога → проблема памяти.

---

## 6. JSON Report Structure (Text Description)
summary — агрегированная информация.  
events — список разобранных событий.  
problems — выявленные проблемы.  
recommendations — рекомендации по устранению.

---

## 7. HTML Report Structure (Text Description)
Блок сводки.  
Таблица событий.  
Блок проблем.  
Блок рекомендаций.

---

## 8. Knowledge Base (MVP)
Медленные SQL → проверить индексы и план запроса.  
Deadlock → анализ конфликтующих транзакций.  
Долгие COMMIT → оптимизация записи.  
Ошибки платформы → проверка конфигурации и кода.

---

## 9. MVP Acceptance Criteria
Обработка 10k строк менее чем за 3 секунды.  
Точность классификации не ниже 90 процентов.  
Корректный JSON и HTML.  
Отсутствие зависимости от конкретной конфигурации 1С.
